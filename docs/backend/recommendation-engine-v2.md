# MaeMoJi 뉴스 감성 엔진 V2

## 처리 순서

1. Finnhub `company-news`로 최근 7일 기사 수집
2. Finnhub 기사 ID 또는 제목과 URL의 SHA-256 해시로 중복 제거
3. 티커와 회사명 토큰으로 1차 종목 관련성 계산
4. 긍정/부정/강한 악재 키워드로 1차 감성 점수 계산
5. Gemini 구조화 JSON으로 기사별 문맥 분석
6. 감성, 관련성, 최신성, 영향도를 이용해 기사별 가중 점수 계산
7. 기사별 결과를 `news_analysis_cache`에 저장
8. 종목별 가중 평균을 추천 엔진의 뉴스 점수로 변환

화면과 DB에는 Gemini 관련성 점수가 60 이상인 기사만 하루 최대 3개 저장한다.
Gemini 호출이 실패하면 기사 감성을 임의로 표시하지 않고 뉴스 점수를 중립 처리한다.

## 기사별 점수

```text
articleWeightedScore =
    sentimentScore
    * (relevanceScore / 100)
    * recencyWeight
    * impactWeight
```

- `sentimentScore`: -100 ~ 100
- `relevanceScore`: 0 ~ 100
- `recencyWeight`: 24시간 이내 1.0, 3일 이내 0.85, 7일 이내 0.65
- `impactWeight`: LOW 0.75, MEDIUM 1.0, HIGH 1.35

종목별 최종 감성 점수는 기사 가중치 합으로 나눈 가중 평균이며 -100 ~ 100 범위를 유지한다.

## 강한 악재 우선 규칙

다음 유형이면서 종목 관련성 점수가 60 이상이면 강한 악재로 처리한다.

- 분식회계 또는 사기
- 파산 또는 Chapter 11
- 상장폐지
- SEC/형사 조사
- 가이던스 하향

강한 악재는 기사 감성 점수를 최대 -85로 제한하고 영향도를 HIGH로 올린다. 종목 전체 뉴스 감성은 최대 -70으로 제한하며, 추천 상태는 최소 `REDUCE`로 조정한다.

## Gemini 응답

Gemini는 `responseJsonSchema`를 사용해 다음 값을 기사별로 반환한다.

```json
{
  "sentimentLabel": "NEGATIVE",
  "sentimentScore": -65,
  "relevanceScore": 90,
  "impactLevel": "HIGH",
  "summaryKo": "향후 실적 가이던스가 낮아졌습니다.",
  "reason": "현재 실적은 양호하지만 가이던스 하향이 향후 주가에 부담이 될 수 있습니다."
}
```

기본 모델은 `gemini-2.5-flash`이며 `GEMINI_MODEL` 환경변수로 교체할 수 있다.

`GET /api/recommendations/news-engine/status`에서 Finnhub/Gemini 환경변수 인식 여부와 현재 모델을 확인할 수 있다. API 키 값 자체는 노출하지 않는다.

## 호출 최적화

Gemini 호출 여부는 단순 시간 캐시가 아니라 다음 조건으로 결정한다.

1. Finnhub 후보 기사 최대 8개의 내용 해시를 정렬한다.
2. 전체 후보를 하나의 `analysis_batch_hash`로 만든다.
3. 기존 DB의 `analysis_batch_hash`와 Gemini 모델명이 모두 같으면 기존 분석을 재사용한다.
4. 기사 내용 또는 모델이 하나라도 바뀌면 Gemini를 다시 호출한다.

캐시를 재사용하더라도 최신성 가중치는 현재 시각을 기준으로 다시 계산한다.
따라서 동일 기사의 LLM 판단은 재사용하지만 시간이 지난 효과는 점수에 반영된다.

로컬 실측에서 AAPL과 GOOGL 두 종목을 연속 생성했을 때 첫 분석은 약 38.88초,
동일 기사 재요청은 약 1.72초였다.

## 점수 안정화

기사 수가 적거나 기사 간 방향이 충돌할 때 극단적인 점수가 나오지 않도록 보정한다.

- 기사 1개: 원점수의 65% 반영
- 기사 2개: 원점수의 85% 반영
- 기사 3개: 원점수의 100% 반영
- 긍정 기사와 부정 기사가 함께 있으면 합의도 계수 80% 적용
- 강한 악재 override는 표본 수 보정과 무관하게 유지

뉴스 분석 신뢰도는 기사 수, 평균 관련성, 기사 간 합의도를 기준으로 0~95 범위에서 계산하며
전체 추천 신뢰도에도 비례 반영한다.

## 정확도 검증

현재 최적화는 호출 중복 제거, 관련성 필터, 구조화 응답, 점수 안정성 관점에서 검증했다.
그러나 감성 판정의 실제 정확도는 별도 평가 데이터가 없으면 수치로 주장할 수 없다.

서비스 품질 검증 단계에서는 최소 100~300개의 기사를 사람이 다음 기준으로 라벨링해야 한다.

- 대상 종목 직접 관련 여부
- 긍정 / 중립 / 부정
- 영향도 LOW / MEDIUM / HIGH
- 강한 악재 여부

이 데이터로 감성 분류 정확도, 관련성 오탐률, 강한 악재 누락률을 측정하고
Gemini와 키워드 혼합 비율을 조정한다.

## 화면 표시

종목 상세의 관련 뉴스 카드에는 다음 정보를 표시한다.

- 감성 라벨과 -100 ~ 100 점수
- 종목 관련성 점수
- 영향도
- 한국어 요약
- 판단 이유
- 뉴스 원문 링크
