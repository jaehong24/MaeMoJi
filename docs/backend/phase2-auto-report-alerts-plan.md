# MaeMoJi Phase 2 기획안

## 주제

자동 주간 리포트 + 추천 변화 알림 + 한 탭 이유 선택

---

## 1. 기획 배경

MaeMoJi는 단순히 종목을 보여주는 앱이 아니라,
사용자가 `왜 이 종목을 모으고 있는지`, `지금 어떤 변화가 생겼는지`,
`다시 확인해야 할 종목이 무엇인지`를 빠르게 이해하게 해주는 서비스가 되어야 한다.

하지만 사용자가 직접 긴 메모를 쓰게 하면
대부분 귀찮아서 입력하지 않거나, 처음만 쓰고 끝날 가능성이 높다.

따라서 다음 단계의 핵심 방향은 아래처럼 잡는다.

1. 사용자가 쓰지 않아도 앱이 먼저 정리해준다
2. 정말 중요한 변화만 알려준다
3. 입력은 자유 메모보다 선택형으로 최소화한다

즉 Phase 2의 핵심은
`기록을 강요하는 앱`이 아니라
`자동으로 요약하고, 중요한 순간에 다시 불러주는 앱`이다.

---

## 2. 이번 Phase의 목표

이번 Phase에서 만들 기능은 아래 3개다.

1. 자동 주간 리포트
2. 추천 변화 알림
3. 한 탭 이유 선택

이 3개는 서로 따로 노는 기능이 아니라 하나의 흐름으로 연결된다.

- 사용자가 종목을 담는다
- 이유를 한 번 탭으로 고른다
- 매모지가 매주 포트폴리오 변화를 자동으로 정리한다
- 변화가 크면 알림으로 다시 부른다

---

## 3. 핵심 사용자 가치

### 3.1 자동 주간 리포트

사용자 입장 가치:

- “이번 주 내 포트폴리오에서 뭐가 바뀌었는지” 앱이 먼저 정리해줌
- 상세 화면을 하나씩 다 열어보지 않아도 됨
- 점수, 뉴스, 가격 흐름 변화의 핵심만 빠르게 확인 가능

### 3.2 추천 변화 알림

사용자 입장 가치:

- 중요한 변화가 생겼을 때만 다시 앱에 들어오게 됨
- 매일 확인하지 않아도 핵심 변화는 놓치지 않음

### 3.3 한 탭 이유 선택

사용자 입장 가치:

- 긴 메모 대신 한 번만 간단히 선택하면 됨
- “왜 담았는지”를 나중에 다시 볼 수 있음
- 추천 결과와 내 의도를 비교할 수 있음

---

## 4. 제품 포지셔닝

이 기능 이후 매모지는 아래처럼 보이게 된다.

- 토스/증권사 앱: 거래와 자산 조회
- 일반 뉴스 앱: 정보 나열
- MaeMoJi: 내 포트폴리오를 자동으로 해석해주는 투자 보조 앱

즉 매모지는
`거래 앱`이 아니라
`해석 앱`
`리마인드 앱`
`투자 변화 요약 앱`
으로 포지셔닝하는 것이 맞다.

## 5. 기능 1: 자동 주간 리포트

## 5.1 목적

사용자가 등록한 포트폴리오 종목에 대해
지난주 대비 이번 주 변화만 요약해서 보여준다.

핵심은 데이터가 많아도
“이번 주 다시 볼 이유가 있는 종목”만 빠르게 보이게 하는 것이다.

## 5.2 사용자 시나리오

1. 사용자가 월요일 아침 또는 일요일 저녁에 앱을 연다
2. 홈 또는 리포트 화면에서 `이번 주 매모지 리포트`를 본다
3. 포트폴리오 전체 변화 요약을 먼저 본다
4. 변화가 큰 종목 3~5개를 우선 본다
5. 관심 종목 상세로 이동한다

## 5.3 리포트 구성

### 상단 요약

- 이번 주 다시 확인할 종목 수
- 증액 의견 증가 종목 수
- 감액/중단 주의 종목 수
- 뉴스 악화 종목 수

### 변화 카드

종목별로 아래를 보여준다.

- 종목명
- 현재 추천 상태
- 지난주 대비 점수 변화
- 핵심 변화 이유 1줄
- 변화 유형 배지

예시:

- 엔비디아
- 증액 유지
- 점수 +6
- 뉴스 개선과 가격 안정이 함께 반영되었어요

### 변화 유형 예시

- 뉴스 개선
- 뉴스 악화
- 가격 과열
- 가격 안정
- 기업 체력 개선 반영
- 데이터 보강 완료

## 5.4 주간 리포트 표시 기준

리포트는 모든 종목을 길게 나열하지 않는다.

우선순위 기준:

1. 추천 상태가 바뀐 종목
2. 점수 변화 폭이 큰 종목
3. 뉴스 분위기 변화가 큰 종목
4. 가격 과열/급락 상태 진입 종목

즉 “이번 주 의미 있는 변화가 있는 종목”만 위로 올린다.

## 5.5 첫 버전에서 보여줄 항목

- 종목별 현재 추천 상태
- 지난주 대비 최종 점수 변화
- 가격 흐름 변화 요약
- 뉴스 분위기 변화 요약
- 핵심 해석 한 줄

## 5.6 첫 버전에서 제외

- 실현 손익 계산
- 브로커 연동 성과 분석
- 주간 수익률 랭킹
- 복잡한 그래프 대시보드

## 6. 기능 2: 추천 변화 알림

## 6.1 목적

매일 모든 종목을 알리지 않고,
정말 다시 봐야 할 변화만 사용자에게 알려준다.

## 6.2 알림이 필요한 상황

### 강한 변화

- `유지 -> 증액`
- `증액 -> 유지`
- `유지 -> 감액`
- `감액 -> 중단`

### 점수 급변

- 점수 7점 이상 상승
- 점수 7점 이상 하락

### 위험 신호

- 최근 뉴스가 강한 악재로 전환
- 가격 과열 구간 진입
- 가격 안정성 급락

### 데이터 상태 변화

- 신규 종목의 가격 흐름 축적 완료
- 재무 데이터 부족 상태 해소

## 6.3 알림 정책

알림은 많이 보내면 피로해지므로 아래 원칙을 둔다.

1. 하루 1회 요약 알림 우선
2. 강한 변화가 없는 날은 보내지 않음
3. 같은 종목 같은 사유로 연속 발송 금지
4. 사용자가 알림 유형을 켜고 끌 수 있게 함

## 6.4 알림 문구 예시

- 엔비디아가 이번 주 다시 볼 종목으로 올라왔어요
- 아마존은 가격 부담이 커져 증액 의견이 유지로 바뀌었어요
- 테슬라는 뉴스 분위기가 악화되어 다시 확인이 필요해요
- 서비스나우는 가격 흐름 데이터가 쌓여 분석 신뢰도가 높아졌어요

## 6.5 알림 화면 연결

알림을 누르면 아래로 이동한다.

- 종목 상세
또는
- 이번 주 리포트 화면 내 해당 종목 위치

첫 버전은 종목 상세 이동이 가장 단순하고 안전하다.

## 7. 기능 3: 한 탭 이유 선택

## 7.1 목적

사용자가 종목을 추가하거나 수정할 때
긴 자유 메모 대신 빠르게 투자 이유를 남기게 한다.

## 7.2 왜 필요한가

추천 결과와 사용자 의도가 다를 수 있기 때문이다.

예:

- 사용자는 장기 성장 때문에 담음
- 매모지는 현재 가격 부담 때문에 유지로 봄

이 차이를 나중에 다시 읽을 수 있어야
매모지가 단순 점수 앱이 아니라 “내 판단을 정리해주는 앱”이 된다.

## 7.3 입력 방식

종목 등록 또는 수정 시 아래 이유 중 1~3개 선택 가능

예시 후보:

- 장기 성장
- 가격 조정 매수
- AI/기술 성장 기대
- 배당/현금흐름
- 우량주 장기 보유
- 분산 투자
- 방어형 자산 보강
- 반등 기대
- 뉴스/이슈 주목

## 7.4 추가 입력 최소화

자유 메모는 선택 사항으로 남기되,
기본 경험은 이유 선택만으로 충분하게 한다.

즉:

- 필수: 선택형 이유
- 선택: 한 줄 메모

## 7.5 활용 방식

선택한 이유는 아래에 활용한다.

1. 종목 상세에서 `내가 담은 이유` 표시
2. 추천 결과와 사용자 의도 비교
3. 주간 리포트에서 의도 대비 변화 안내
4. 나중에 공개 프로필/커뮤니티로 확장 가능

예시:

- 내가 담은 이유: 장기 성장, AI/기술 성장 기대
- 현재 매모지 해석: 기업 체력은 강하지만 가격 부담으로 증액은 보수적으로 봐요

## 8. 사용자 흐름

## 8.1 종목 등록 시

1. 사용자가 종목 검색
2. 종목 추가
3. 매일 모으기 금액 입력
4. 이유 1~3개 탭 선택
5. 저장
6. 백엔드가 추천 계산 및 워밍

## 8.2 평소 사용 시

1. 사용자가 앱 진입
2. 홈에서 이번 주 리포트 요약 확인
3. 변화 큰 종목 상세 진입
4. 내 이유와 매모지 해석 비교

## 8.3 알림 시

1. 사용자가 푸시/인앱 알림 수신
2. 해당 종목 상세 또는 주간 리포트 이동
3. 변화 이유 확인

## 9. 화면 기획

## 9.1 홈 화면

기존 홈 상단 또는 중간에
`이번 주 매모지 리포트` 섹션 추가

표시 요소:

- 이번 주 다시 볼 종목 수
- 변화 요약 카드 2~3개
- 전체 리포트 보기 버튼

## 9.2 리포트 전용 화면

화면명 예시:

- 이번 주 리포트
- 이번 주 매모지

구성:

1. 상단 요약
2. 변화 큰 종목 목록
3. 경고/주의 종목 구역
4. 긍정 전환 종목 구역

## 9.3 종목 등록/수정 화면

이유 선택 섹션 추가

표시 형태:

- 칩 선택형
- 최대 3개 선택

## 9.4 종목 상세 화면

기존 추천 근거 아래 또는 위에
`내가 담은 이유` 카드 추가

예시:

- 내가 담은 이유
- 장기 성장
- AI/기술 성장 기대

## 10. DB 설계 초안

## 10.1 신규 테이블 1: `portfolio_item_reasons`

역할:

- 포트폴리오 항목별 투자 이유 저장

컬럼 초안:

- `id`
- `portfolio_item_id`
- `reason_code`
- `display_order`
- `created_at`

제약:

- `unique(portfolio_item_id, reason_code)`

## 10.2 reason_code 후보

- `LONG_TERM_GROWTH`
- `PRICE_PULLBACK`
- `AI_GROWTH`
- `DIVIDEND`
- `QUALITY_CORE`
- `DIVERSIFICATION`
- `DEFENSIVE_BALANCE`
- `REBOUND_EXPECTATION`
- `NEWS_EVENT`

## 10.3 신규 테이블 2: `portfolio_weekly_reports`

역할:

- 사용자별 주간 리포트 메타 저장

컬럼 초안:

- `id`
- `user_id`
- `report_week`
- `generated_at`
- `headline`
- `summary`
- `changed_item_count`
- `alert_item_count`
- `positive_item_count`
- `negative_item_count`

제약:

- `unique(user_id, report_week)`

## 10.4 신규 테이블 3: `portfolio_weekly_report_items`

역할:

- 종목별 주간 변화 저장

컬럼 초안:

- `id`
- `report_id`
- `portfolio_item_id`
- `stock_id`
- `current_status`
- `previous_status`
- `score_delta`
- `headline_label`
- `change_type`
- `summary`
- `display_order`

## 10.5 신규 테이블 4: `user_alert_events`

역할:

- 사용자에게 보낼 변화 이벤트 저장

컬럼 초안:

- `id`
- `user_id`
- `portfolio_item_id`
- `stock_id`
- `alert_type`
- `title`
- `body`
- `dedupe_key`
- `sent_at`
- `read_at`
- `created_at`

제약:

- `unique(dedupe_key)`

## 11. API 설계 초안

## 11.1 이유 선택 저장

### `PUT /api/portfolio/items/{portfolioItemId}/reasons`

요청 예시:

```json
{
  "reasons": [
    "LONG_TERM_GROWTH",
    "AI_GROWTH"
  ]
}
```

응답:

- 저장된 reason 목록

## 11.2 이유 조회

### `GET /api/portfolio/items/{portfolioItemId}/reasons`

응답:

- 선택 이유 목록

## 11.3 이번 주 리포트 조회

### `GET /api/reports/weekly/latest`

응답:

- 주차
- 상단 요약
- 변화 종목 리스트

## 11.4 과거 리포트 목록

### `GET /api/reports/weekly`

응답:

- 최근 주간 리포트 목록

## 11.5 알림 목록

### `GET /api/alerts`

응답:

- 읽지 않은 알림
- 최근 알림

## 11.6 알림 읽음 처리

### `POST /api/alerts/{alertId}/read`

## 12. 배치/백엔드 처리 구조

## 12.1 주간 리포트 생성 시점

권장:

- 매주 일요일 저녁 또는 월요일 새벽

이유:

- 미국장 주간 흐름 반영이 쉬움
- 사용자가 월요일 시작 전에 리포트 보기 좋음

## 12.2 생성 흐름

1. 사용자별 활성 포트폴리오 조회
2. 최신 추천 결과와 지난주 결과 비교
3. 상태 변화 / 점수 변화 / 뉴스 변화 계산
4. 핵심 종목 선별
5. `portfolio_weekly_reports`, `portfolio_weekly_report_items` 저장
6. 알림 이벤트 생성

## 12.3 알림 이벤트 생성 기준

- 상태 변화
- 점수 급변
- 강한 악재
- 가격 과열 진입

## 14. 성공 지표

이번 기능이 성공했는지 보는 기준:

1. 사용자가 주간 리포트를 실제로 열어보는가
2. 알림 클릭 후 상세 화면 진입률이 높은가
3. 종목 등록 시 이유 선택 완료율이 높은가
4. 포트폴리오 재방문율이 증가하는가

핵심은 “많이 쓰게 하는 기능”보다
“다시 들어오게 하는 기능”인지다.

## 15. 최종 결론

MaeMoJi의 다음 기획은 거래 연동보다
`자동 요약`, `변화 감지`, `최소 입력`에 집중하는 것이 맞다.

이번 Phase의 핵심은 아래 한 줄로 정리된다.

`매모지가 먼저 정리해주고, 중요한 변화만 알려주고, 사용자는 한 번 탭으로 의도를 남긴다.`

이 구조가 자리 잡으면,
그 다음에 공개 프로필, 팔로우, 커뮤니티, 투자자 피드 같은 확장도 자연스럽게 이어질 수 있다.
