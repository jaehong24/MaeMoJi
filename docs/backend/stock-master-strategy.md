# MaeMoJi 상위 500 종목 적재 전략

## 목표

MaeMoJi는 종목 검색 속도와 추천 엔진 일관성이 중요합니다. 그래서 자주 검색되는 미국 대형주와 ETF는 외부 API를 매번 직접 조회하기보다 `stocks` 테이블에 미리 적재해 두는 전략이 효율적입니다.

## 권장 범위

- 1차 범위: 미국 시가총액 상위 500 종목
- 포함 대상: 대형주, 대표 ETF, 사용자가 자주 찾는 지수 추종 ETF
- 제외 대상: 거래정지 종목, 중복 심볼, 유동성 낮은 예외 종목

## 왜 미리 저장하는가

- 검색 속도가 빨라집니다.
- 한글명, 영문명, 거래소, 로고를 서비스 기준으로 정리할 수 있습니다.
- 포트폴리오 등록과 추천 엔진이 동일한 `stock_id`를 기준으로 움직여 데이터가 안정됩니다.
- 외부 API 호출 수를 줄여 비용과 장애 영향을 낮출 수 있습니다.

## 저장 기준 컬럼

`stocks` 테이블에는 아래 정보를 유지합니다.

- `ticker`
- `exchange_code`
- `finnhub_symbol`
- `name_ko`
- `name_en`
- `logo_url`
- `market_type`
- `is_active`

추가로 추천 엔진용 빠른 조회를 위해 별도 스냅샷 테이블에 아래 값을 저장하는 확장을 권장합니다.

- `market_cap`
- `current_price`
- `change_rate_1d`
- `change_rate_30d`

## 동기화 전략

### 1. 초기 적재

- Finnhub 또는 별도 시총 기준 데이터 소스에서 상위 500 목록 확보
- `stocks`에 upsert
- 한국 사용자 친화적인 대표 한글명은 별도 매핑으로 보강

### 2. 정기 갱신

- 하루 1회 장 마감 후 갱신 권장
- 변경 대상:
  - 회사명 변경
  - 로고 변경
  - 거래소 코드 변경
  - 비활성 종목 여부

### 3. 온디맨드 보강

- 상위 500 밖 종목은 사용자가 검색했을 때만 외부 API 조회
- 조회 후 유효 종목이면 `stocks`에 upsert

## 운영 권장안

- `상위 500~1000`은 DB 캐시 중심
- 그 외 종목은 실시간 API 조회 후 필요 시 저장
- 추천 엔진은 반드시 `stocks.id`를 기준으로 포트폴리오와 연결

## MaeMoJi 현재 단계 권장 결론

현재 단계에서는 아래 순서가 가장 현실적입니다.

1. `stocks`에 샘플 종목과 대표 대형주를 먼저 유지
2. Flutter 검색은 WAS의 `stocks` 조회를 사용
3. 이후 배치 작업으로 상위 500 자동 적재 추가
4. 추천 엔진은 `portfolio_items.stock_id` 기준으로 확장
