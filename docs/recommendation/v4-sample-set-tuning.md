# MaeMoJi V4 샘플셋 튜닝 기준

## 목적

`VALUATION`, `QUALITY_OF_GROWTH`, `FUNDAMENTAL_QUALITY`가 실제 사용자 체감과 맞는지 점검하기 위한 대표 종목 묶음입니다.

점수모델을 조정할 때는 아래 샘플셋을 기준으로

- 점수 포화가 생기지 않는지
- 밸류에이션이 실제로 분리되어 보이는지
- 성장의 질이 매출 성장률만으로 과대평가되지 않는지
- 같은 대형주라도 성격 차이가 결과에 반영되는지

를 함께 확인합니다.

## 기대 관찰 포인트

### 1. 초대형 성장주 내부 서열

- `NVDA`는 매우 높은 성장성과 수익성으로 상단권이어야 합니다.
- `GOOGL`은 `NVDA`보다 약간 낮지만 상위권이어야 합니다.
- `AMZN`은 성장성은 좋지만 수익성/마진 구조 때문에 `GOOGL`보다 한 단계 낮게 나와야 합니다.

### 2. 품질주와 성장주의 차이

- `COST`, `MCD`, `PG`는 성장률이 아주 높지 않아도 안정적 품질주로 평가되어야 합니다.
- 다만 `NVDA`, `GOOGL`과 거의 같은 점수로 붙어버리면 포화로 봅니다.

### 3. 밸류에이션 분리 확인

- `TSLA`, `PLTR`, `ARM`은 기업 체력만 보면 장점이 있어도
  `VALUATION` 팩터에서 부담이 보여야 합니다.
- 사용자는 상세에서 “좋은 기업이지만 지금 가격은 비싸다”를 읽을 수 있어야 합니다.

### 4. 성장의 질 확인

- `QUALITY_OF_GROWTH`는 매출 성장만 보지 않습니다.
- `매출 성장 + EPS 흐름 + 영업이익률 + 현금흐름 품질`을 함께 봅니다.
- 따라서 매출만 빠르게 늘고 현금흐름이 약한 종목은 과대평가되면 안 됩니다.

## 운영 튜닝 기준

### VALUATION

- PER만 높다고 무조건 최하점으로 몰지 않습니다.
- 다만 `ATTRACTIVE / FAIR / EXPENSIVE / VERY_EXPENSIVE` 구간이 사용자에게 납득 가능해야 합니다.

### QUALITY_OF_GROWTH

- 높은 매출 성장률이 있어도
  - EPS가 약하거나
  - 영업이익률이 낮거나
  - 현금흐름 품질이 나쁘면
  최종 점수는 한 단계 보수적으로 내려가야 합니다.

### FUNDAMENTAL_QUALITY

- 수익성
- 재무 안정성
- 현금흐름
- 자산 효율

이 네 축이 중심이어야 하며, `VALUATION`은 별도 팩터로 분리된 상태를 유지합니다.
